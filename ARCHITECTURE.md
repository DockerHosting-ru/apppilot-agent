# Архитектура AppPilot Agent

## Обзор

AppPilot Agent - это Python-приложение, которое работает как агент для платформы AppPilot. Агент отвечает за развертывание и управление Docker контейнерами на сервере.

## Компоненты

### 1. Основной класс: ApplikuAgentCompose

Основной класс агента, который управляет:
- Подключением к Docker API
- Коммуникацией с AppPilot API сервером
- Развертыванием приложений
- Управлением жизненным циклом контейнеров

### 2. Docker Compose поддержка

Агент поддерживает развертывание multi-container приложений через Docker Compose:
- Автоматическое создание docker-compose.yml файлов
- Управление зависимостями между сервисами
- Автоматическое назначение портов

### 3. Шаблоны приложений

Предустановленные шаблоны для популярных приложений:
- **n8n + PostgreSQL** (template_id: 100)
- **WordPress + MySQL** (template_id: 101)
- **Nextcloud + PostgreSQL** (template_id: 102)

## Архитектура развертывания

```
AppPilot API Server → Agent → Docker Engine → Containers
```

1. **API Server** отправляет команды развертывания
2. **Agent** получает команды и выполняет их
3. **Docker Engine** создает и управляет контейнерами
4. **Containers** запускаются с назначенными портами

## Управление портами

Агент автоматически назначает уникальные порты для каждого приложения:
- Диапазон: 8001-9000
- Проверка доступности порта через socket.bind()
- Предотвращение конфликтов портов

## Аутентификация

Используется JWT токен для безопасной коммуникации с API сервером:
- Bearer токен в заголовке Authorization
- X-Agent-ID заголовок для идентификации агента

## Мониторинг

Агент предоставляет:
- Логирование всех операций
- Статус выполнения задач
- Мониторинг состояния контейнеров
- Обработка ошибок и восстановление

## Конфигурация

Настройка через переменные окружения:
- `API_SERVER_URL` - URL API сервера
- `AGENT_ID` - Уникальный ID агента
- `JWT_TOKEN` - Токен аутентификации
- `POLL_INTERVAL` - Интервал опроса сервера
- `PORT_RANGE_START/END` - Диапазон портов
